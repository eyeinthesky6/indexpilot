## Executive Summary

This document analyzes the licensing terms for academic algorithms identified in our research and provides guidance on commercial use in IndexPilot. **IndexPilot uses MIT License**, which is compatible with most open-source licenses but requires careful review of academic paper licenses.

**Key Finding**: Most algorithms can be used commercially with proper attribution, but each paper/implementation must be checked individually.

---

## Licensing Overview

### IndexPilot's License
- **License**: MIT License
- **Commercial Use**: ✅ Allowed
- **Attribution Required**: ✅ Yes (copyright notice)
- **Compatibility**: Compatible with most open-source licenses

### Academic Paper Licensing (arXiv/TechRxiv)

**Common License Types:**
1. **CC BY 4.0 (Creative Commons Attribution)**: ✅ Commercial use allowed with attribution
2. **CC BY-SA 4.0 (Attribution-ShareAlike)**: ✅ Commercial use allowed, but derivatives must use same license
3. **CC BY-NC 4.0 (Attribution-NonCommercial)**: ❌ Commercial use NOT allowed
4. **CC BY-NC-ND 4.0 (NonCommercial-NoDerivatives)**: ❌ Commercial use NOT allowed
5. **arXiv Default License**: ⚠️ Need author permission for commercial use
6. **CC0 1.0 (Public Domain)**: ✅ No restrictions, no attribution required

## Recommended Approach

### Phase 1: License Verification (Before Implementation)

1. **Check Each Paper's License**:
   - Visit arXiv page for each paper
   - Look for license information (usually in metadata)
   - Document license type

2. **Check for GitHub Implementations**:
   - Search GitHub for implementations
   - Check implementation license (often MIT/Apache)
   - Prefer implementations over re-implementing from papers

3. **Contact Authors if Needed**:
   - If license is unclear or restrictive
   - Request commercial use permission
   - Document permission in writing

### Phase 2: Implementation Strategy

**Option A: Use Existing Implementations (Recommended)**
- Find open-source implementations on GitHub
- Check license compatibility (MIT/Apache compatible with MIT)
- Use implementation directly or as reference
- **Advantage**: Clear licensing, tested code

**Option B: Implement from Papers**
- If no implementation available
- Verify paper license allows commercial use
- Implement algorithm based on paper
- Provide proper attribution

**Option C: Inspired Implementation**
- Implement similar concepts (not exact algorithm)
- No license issues (algorithms themselves aren't copyrightable)
- Still provide attribution as good practice
- **Advantage**: No licensing concerns

### Phase 3: Attribution Requirements

**If Commercial Use is Allowed:**

1. **Code Comments**:
```python
# Based on "The PGM-index: a fully-dynamic compressed learned index"
# by [Authors], arXiv:1910.06169
# Licensed under CC BY 4.0 (or appropriate license)
```

2. **Documentation**:
- Add to README.md or docs
- List all academic sources
- Include paper links and licenses

3. **License File**:
- Create `THIRD_PARTY_LICENSES.md` or similar
- List all academic sources
- Include full attribution

## Legal Recommendations

### ✅ Safe Practices

1. **Always Check License**: Never assume - always verify
2. **Use Implementations**: Prefer GitHub implementations with clear licenses
3. **Provide Attribution**: Always credit original authors
4. **Document Sources**: Keep records of all sources and licenses
5. **Consult Legal**: When in doubt, consult legal counsel

### ⚠️ Risky Practices

1. **Assuming CC BY**: Not all arXiv papers are CC BY
2. **Ignoring Attribution**: Even if allowed, attribution is good practice
3. **Using Non-Commercial**: CC BY-NC does NOT allow commercial use
4. **No Documentation**: Failing to document sources creates risk

### ❌ Prohibited Practices

1. **Using CC BY-NC Commercially**: Violates license terms
2. **No Attribution**: Violates CC BY terms
3. **Ignoring ShareAlike**: CC BY-SA requires same license for derivatives
